三、Design

苹果（Apple）

📱 WWDC 2025 发布全新设计语言 Liquid Glass，统一全生态交互体验
图片来源：Apple WWDC 2025

概要： Apple在2025年WWDC上发布全新设计语言Liquid Glass，这是一种“动态数字超材质”，通过动态光学效果、自适应系统与跨平台适配逻辑，统一iPhone、Mac、visionOS等全生态设计语言，实现“轻盈透明却可交互”的视觉与触觉革新，助力开发者打造更贴合用户直觉的App与系统界面。

详情如下：

  • 核心定位：并非单一视觉效果，而是融合Aqua、实时模糊、灵动岛等经典设计经验的“数字超材质”，以“动态折射光线、模拟液体特性”为核心，平衡统一设计与体验生动性，让用户聚焦内容本身。
  • 动态属性：①透镜效果实现分层显示，下层内容可见且与上层控件分离；②元素显隐通过光线折射/透射完成，无需淡入淡出；③触摸时触发“倒映+光线加强”“凝胶弹性”反馈，仅交互时激活效果，静态保持简洁。
  • 自适应性：①分层动态调整（阴影、着色、材质厚度随内容/尺寸变化），确保控件清晰不遮挡；②跨平台适配iPad/Mac等，以悬浮控件/导航层形式预留大内容画布，细节上保持圆角同心度、光线溢出强化高度感；③与滚动边缘效果协同，动态适配背景深浅，特殊场景启用“硬风格”视觉分离。
  • 应用原则：①分层系统（强光层+阴影层+交互发光层）协同工作，优先用于导航层，禁止多元素叠放；②含常规变体（全功能自动适配）与透明变体（高透明需搭配调暗层，仅适用于媒体丰富内容）；③小元素随背景切换外观，大元素保持稳定，着色仅突出核心操作，支持系统级辅助功能适配（降低透明度、提高对比度等）。
  • 配套资源：提供《Adopting Liquid Glass》实操指南与Human Interface Guidelines（Materials模块），助力开发者落地应用。

重要性： Liquid Glass突破了传统数字材质“复刻现实”的局限，解决了“透明与可辨性”的核心矛盾，首次实现Apple全生态设计语言统一；其动态光学与自适应逻辑，不仅提升了人机交互的自然度与质感，更为开发者提供了清晰的设计准则，推动Apple生态界面体验进入“流体动态”新阶段，进一步强化生态一致性优势。

四、有意思

Google Developers Blog — Introducing A2UI: An open project for agent-driven interfaces

谷歌开发者博客 —— 推出 A2UI：一个面向代理驱动界面的开源项目

概要：

Google 正式发布了 A2UI（Agent-to-User Interface）这一开源项目，旨在为生成式 AI 代理创建 可跨平台、安全、原生渲染的用户界面规范，使 AI 不再仅通过文本或命令行交互，还能生成富 UI 以提升交互体验。该项目当前处于早期公开阶段（v0.8）。(Google 开发者博客)

详情如下：

  • 项目定义与目标： A2UI 是一个开源协议规范，使 AI 代理能够生成结构化的 UI 描述（JSON 格式），由前端应用使用自身原生组件库渲染，而非执行任意代码，从而解决跨信任边界的 UI 生成问题。(Google 开发者博客)
  • 用户体验挑战： 传统文本对话模式在复杂任务（例如订餐）中交互效率低下；A2UI 允许代理生成动态表单、时间选择器等界面组件，提升交互效率和直观性。(Google 开发者博客)
  • 安全设计原则： A2UI 是声明性数据格式（非代码），前端保留对组件目录的控制，从而避免 UI 注入风险，确保安全。(Google 开发者博客)
  • 框架与跨平台支持： 该协议框架中立，可在不同 UI 框架（如 Lit、Angular、Flutter、React、SwiftUI 等）上渲染；渲染端负责本地样式和交互实现。(Google 开发者博客)
  • 生态集成： Google 提到 A2UI 可与 AG UI、A2A 协议、Flutter 的 GenUI SDK、Opal、Gemini Enterprise 等集成，共同支撑复杂、多代理场景的 UI 交互。(Google 开发者博客)
  • 协作与开源： 项目采用 Apache 2 许可，欢迎社区贡献，包括拓展渲染器、实现更多客户端支持和样例等。(A2UI)

重要性：

A2UI 的发布标志着 AI 代理交互方式从文本驱动向 UI 协作驱动的实质性转变。它为构建安全、可扩展、原生感强的 AI 交互界面提供了开放标准，有望成为多代理系统和跨组织协作的基础组件规范，提升用户体验和开发效率。(Google 开发者博客)

苹果将在未来的 AirPods Pro 中扩展健康监测功能，新增磨牙症检测功能

图片来源：Patently Apple

概要： 苹果公司获得美国专利，通过 AirPods Pro 内置运动传感器（加速计）结合机器学习技术诊断和监测磨牙症，无需侵入式设备，标志其健康监测功能向行为诊断领域拓展。

详情如下：

  • 技术原理：利用高带宽加速计捕捉磨牙产生的骨传导振动信号，经分割、滤波后，通过支持向量机等机器学习分类器区分磨牙与说话、进食等活动。
  • 核心优势：非侵入式设计（无需面部电极）、支持睡眠/清醒状态实时监测、依托现有设备（用户无需额外购置）、抗环境噪音干扰，可评估磨牙严重程度并长期追踪。
  • 应用场景：将与 Apple Health 集成，支持用户查看磨牙症长期趋势，必要时可向临床医生共享数据，提前规避牙齿损伤和慢性疼痛风险。
  • 专利细节：发明人为苹果机器智能团队 AI/机器学习研究工程师 Fereshteh Shahmiri 博士，专利含耳塞使用示意图、系统框图、声学特征频谱图三类图示。

重要性： 该专利进一步完善了苹果可穿戴设备健康生态，从传统健身、心率监测延伸至预防保健与行为诊断领域，强化其“全面健康平台”定位，也体现了苹果以专利为基础、持续拓展设备功能边界的产品策略。

七、Car

通往能像人类一样思考和行动的自动驾驶汽车之路

📱The road to self-driving cars that think and behave like humans

图片来源：Autocar

概要： 2025年12月23日，博世与大众集团软件部门Cariad（二者于2022年组建自动驾驶联盟）宣布合作开发具备“认知能力”的自动驾驶技术，目标打造能像人类驾驶员一样思考和行动的自动驾驶系统，其基于AI的软件栈正处于车队测试阶段，计划2026年年中实现量产，以提升L2、L3级自动驾驶的性能与安全性。

详情如下：

  • 合作背景与目标：双方组建的自动驾驶联盟初衷是开发L2级（城市道路、高速公路、乡村道路双手脱离驾驶）与L3级（高速公路完全自主控制）自动驾驶系统，此次进一步聚焦“类人驾驶”，旨在让系统能像人类一样分析场景、推理判断、做出关键决策，同时完成精准定位、辨别行驶方向等基础任务。
  • 核心技术与方案：采用“基于AI的软件栈”，可处理感知、解读、决策、行动等所有核心认知任务；当前软件已为后续升级奠定基础，未来计划整合“视觉-语言-行动”方案，模仿人类逻辑思维与行动模式，以应对复杂交通场景（如识别潜在隐藏风险）。
  • 技术优势与迭代逻辑：通过持续输入海量数据，利用AI擅长数据处理的特性，提升L2、L3级自动驾驶系统的稳定性；软件架构设计注重安全性、可追溯性与可解释性，确保AI决策与行动可追责，避免“黑箱操作”。
  • 发展进度：该AI软件栈目前已在车队中开展测试，按计划将于2026年年中完成开发并投入量产。

重要性： 此次合作聚焦的“认知型”自动驾驶技术，突破了传统自动驾驶依赖规则化编程的局限，通过AI模拟人类驾驶的感知与决策逻辑，有望解决复杂交通场景下的长尾问题，推动L2、L3级自动驾驶向更安全、更贴近人类驾驶习惯的方向发展；同时，其对软件架构“可解释性”与“可追责性”的重视，也为行业在自动驾驶安全规范与技术标准制定上提供了参考，助力自动驾驶技术从“功能实现”向“安全落地”迈进。
